Das Logbuch

August 2020


04.08.20

Erster Eintrag

  • Bisher habe ich Atom als IDE genutzt und bin heute auf VS Code umgestiegen. Ich finde es schon nach ein paar Minuten deutlich besser, umfangreicher und Benutzerfreundlicher.

  • Gleichzeitig versuche ich gerade mich in Git und Github einzuarbeiten, um zum einen von mehreren Geräten aus an diesem Blog arbeiten zu können und zum anderen später besser nachvollziehen zu können, wie stümperhaft ich gerade arbeite. Außerdem habe ich den Blog bereits über Github Pages veröffentlicht.

  • Leider gibt es genau hierbei ein Problem. Einige Styling-Eigenschaften, die offline funktionieren, scheinen auf der Webpage nicht angewendet zu werden. So sind dieSocial-media-icons auf der Startseite nicht mehr zentriert, sndern linksbündig, die Links im Footer haben ihr Syling verloren und das Favicon wird auch nicht angezeigt.


05.08.20

Github und Bootstrap

  • Habe es geschafft Github mit VS Code zu verbinden. Sehr hilfreich fand ich hierbei folgende Videos:
    • Visual Studio Code | How to use git and github
    • How To Use GitHub with VS Code in 2020 | Clone | Part 2
    • Und für speziell für Linux: How to Install and Configure Git and GitHub on Ubuntu 18.04/ Ubuntu 20.04(Linux)
  • Aktuelle Probleme, die ich bisher identifiziert habe:
    • Layout auf dem Smartphone (verzerrte Bilder, Überschriften sind hinter der Navbar)
    • Die Icons auf der Startseite
    • Lange Ladezeiten für Bilder
  • Kein Problem, was mich aber stört:
    • Wenn ich etwas im Footer oder in der Navbar ändere, muss ich das noch auf jeder Seite einzeln tun
    • Ich bin mir noch nicht sicher, wie ich die Ordner und Seiten strukturieren soll. Bisher habe ich home, css, images, blogtopichs (Themenseiten) und blogposts (für die Blogbeiträge)
    • Mein styles.css ist recht unübersichtlich und sollte besser geordnet werden
    • Ich benutze kaum Tags wie section, article, figurecaption usw., weiß aber auch nicht, wie sinnvoll oder hilfreich diese sind
  • Konnte das Problem mit den Icons dadurch lösen, dass ich den Link zu styles.css im head unter den Bootstrap-Link gesetzt habe. Die Links im Footer ist auf allen Seiten außer Index noch immer blau statt hellgrau.
  • Überlege nun statt Bootstrap mit dem CSS Grid System und Flexbox zu arbeiten. Dann sollten die Macken verschwinden. Allerdings ist das auch etwas aufwendig.

06.08.20

Bye, Bye, Bootstrap

  • Versuche mich heute stärker mit dem CSS Grid vertraut zu machen. Auf dieser Seite gibt es einen ganz guten Überblick dazu.
  • Habe nun Header/Navbar und Footer selbst erstellt und komme ohne Bootstrapaus. Die Aktion hat mir dabei natürlich sämtliches Stlying auf allen Seiten zerschossen und ich bin heute und morgen durchgehend damit beschäftigt, das wieder hin zu bekommen, nur damit es exakt so aussieht wie vorher. Sehr motivierend.
